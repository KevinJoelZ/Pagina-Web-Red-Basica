Verificación de Conectividad

Pruebas con ping, traceroute y comandos de diagnóstico para confirmar que la red opera correctamente.

En este módulo

  • Introducción
  • Pruebas con ping
  • Comandos de diagnóstico

¿Por qué verificar?

Configurar una red no garantiza que funcione. La verificación sistemática detecta errores de IP duplicada, máscara incorrecta, gateway mal configurado o cables mal conectados.

Pruebas recomendadas

  1. Ping del PC0 al PC1 (misma subred).
  2. Ping de cada PC al gateway (192.168.1.1).
  3. Revisar tabla ARP en cada PC.
  4. Usar modo simulación para ver el recorrido de paquetes.
Resultado esperado: Todas las pruebas de ping deben responder con Reply from... y 0% packet loss.
